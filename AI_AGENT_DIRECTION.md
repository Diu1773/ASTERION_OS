# ASTERION AI Agent — 기능 틀 & 방향 (§10/§12)

> **갱신 2026-06-23.** 차별점·안전 원칙은 유지하고, *구현 전 북극성*이던 이 문서를 **현재 구축분 +
> 능력 사다리(L0–L5) + 갭 + 우선순위**로 다시 잡았다. 한 줄 요약: **AI는 이미 해석·계획·(승인)실행·
> 학습까지 와 있다. "화성 보여줘 수준"으로 느껴지는 건 능력이 아니라 *체감*(공급자/모델·반응형·기억
> 없음)의 문제다.** 똑똑해지는 다음 층은 L5(능동·조건부·기억)이다.
>
> 근거: deep-research(2026-06-16). 자율 관측소의 *기본기*(스케줄링·안전·품질평가)는 이미 표준이라
> 차별화가 안 된다. 빈틈 = **(1) 범용 대화 제어 + (2) 운영자 피드백 학습.** 선례 StarWhisper
> Telescope(NINA에 LLM 덧붙임·초신성 한정·function-call 70%)가 개념을 입증했고, ASTERION은 더
> 깨끗한 기반(ActionBus·온톨로지)을 가졌다.

---

## 1. 불변 원칙 (이건 안 바뀐다)

1. **AI에게 새 권한을 주지 않는다.** AI는 사람/대시보드와 **똑같은 API만**, `ActionBus`를 통과해
   호출한다 → fail-closed 안전 게이트가 AI에게도 그대로(주간 슬루·악천후 촬영 *불가*). **별도 AI
   안전장치 불필요** — 이게 결정적 이점. 모든 AI 행동은 `ActionLog`/`Decision`에 감사 기록.
   → 남은 리스크는 *위험*이 아니라 **정확도**(SWT 70% 문제). "똑똑하게" = 모델에 직접 제어권을
   더 주는 게 아니라, **더 좋은 도구 + 능동성 + 기억**을 주는 것이다.
2. **두꺼운 도구 / 얇은 모델.** 어려운 일은 코드(astropy·Orchestrator·Sentinel·스케줄러)가 하고,
   모델은 **라우팅 + 설명**만. 그래서 약한 모델로도 되고, 파인튜닝 불필요. (구조가 Palantir AIP와
   1:1 — 온톨로지 + 게이트된 액션 + 벤더중립 모델.)
3. **차별점 = 대화제어 + 피드백학습 + 청람 수직통합.** "또 하나의 NINA"가 아니다. 기본기(L1–L4)는
   table-stakes, **해자(moat)는 L5**(능동·조건부 자율·기억)다.

---

## 2. 능력 사다리 (L0–L5) — 지금 우리가 선 곳

| 레벨 | 정체성 | 무엇을 하나 | 도구(현재) | 상태 |
|---|---|---|---|---|
| **L0** | 음성 리모컨 | "화성 보여줘" → 슬루 (단발 명령→행동) | `goto_planet` | ✅ (사용자가 *보는* 전부) |
| **L1** | **해석자** | "왜 unsafe야"·"어젯밤 어땠어"·"마운트 왜 이래" → 상태·이력 설명 | `get_status`·`diagnose`·`night_report`·`telemetry_summary`·`planet_position` | ✅ 구축 |
| **L2** | **계획자** | "오늘 밤 은하로 일정 짜줘"·"메시에 채우기 목표" → 시간표·캠페인 생성 | `plan_night`·`set_goal`·`create_plan` | ✅ 구축 |
| **L3** | **운영자** | "오늘 밤 자동으로 돌려" → 승인된 시간표 무인 실행/정지 | `run_night`·`stop_night`·`run_plan`·`dome_shutter` | ✅ 구축 (= 차별점①) |
| **L4** | **학습자** | "M13 어떻게 찍는 게 좋아" → 품질·SNR 분석→추천, Decision에 기록→다음 계획 반영 | `target_feedback` | 🟡 부분 (루프는 닫힘, 얕음) |
| **L5** | **협력자(co-pilot)** | 먼저 말하고·조건을 기다리고·과거를 기억 | — | ❌ **갭 = 다음 층** |

**핵심: 실제로는 L1–L4까지 와 있다.** L0(음성 리모컨)은 빙산의 일각이고, "똑똑함"의 체감은 L5에서 나온다.

---

## 3. 왜 "화성 보여줘 수준"으로 느껴지나 (정직한 진단)

능력은 L4까지인데 *체감*이 L0인 건 세 가지 때문이다 — 전부 능력이 아니라 **체감/배선** 문제:

1. **공급자·모델 문제(1순위 의심).** 채팅은 `config.local.json`에 키가 있어야 돌고, 기본 모델
   `llama-3.3-70b`은 **tool-call을 간헐적으로 망가뜨린다**(Groq 400). 도구호출이 실패하면 복잡한
   요청(plan_night·diagnose)이 안 먹히고 *제일 단순한 것만* 되는 것처럼 보인다 → "화성 보여줘밖에
   안 됨"의 정체가 이것일 가능성이 높다. **해결: 도구호출 강한 모델**(`openai/gpt-oss-120b`·
   `qwen/qwen3-32b`)로 스왑.
2. **반응형이라 먼저 말을 안 한다.** 사용자가 *마법의 문구*를 알아야 하고, AI가 먼저 "지금 GoTo가
   3연속 실패했어요, 닫을까요?"라고 *나서지* 않는다. 진짜 "비서 같은" 느낌은 능동성에서 온다.
3. **기억이 없다.** 매 대화가 백지다. "지난번처럼 해줘"가 안 된다 — 과거 Decision/Feedback을
   대화 컨텍스트로 안 끌어온다.

---

## 4. 갭 = L5 (똑똑함이 사는 곳)

해자는 여기다. 다섯 조각, 전부 **읽기 위주거나 ActionBus 게이트 안**이라 안전면은 작다:

- **(a) 능동 내레이션** — `EventHub` 리스너가 이상징후를 AI가 *먼저* 한 줄로: "GoTo 3연속 실패 +
  기상 stale → 닫기 권고." 인프라(EventHub·toolkit)는 이미 있고 배선만. **"살아있다" 체감 최대 레버.**
- **(b) NL 로그/이벤트 검색** — "지난주 돔 안 닫힌 적 있어?", "어젯밤 왜 멈췄어?" 자유질의. §12.2가
  명시한 능력인데 미구현(`diagnose`가 고정창만 커버). `ActionLog`/`Alert`/`Decision`에 시간·텍스트
  필터 도구 하나. 읽기전용·안전면 0 → **가장 자연스러운 다음 한 수.**
- **(c) 조건부 자율(트리거 엔진)** — "구름 걷히면 찍어"·"습도 80 밑으로 떨어지면 시작." 안전 스냅샷을
  구독하는 트리거 평가기. (5단 로드맵의 3단.) 자율의 진짜 다음 계단.
- **(d) 대화 기억(RAG-over-온톨로지)** — 과거 `Decision`/`Feedback`/`QualityMetric`을 검색해 컨텍스트
  주입("이 대상은 지난번 60s가 과했음"). L4를 얕은 단발 추천 → *맥락 있는 조언*으로 깊게.
- **(e) 음성(STT/TTS)** — 돔 안에서 손 안 쓰고. 폴리시, 후순위.

---

## 5. 우선순위 (레버리지 / 난이도)

1. **공급자·모델 점검** (한 줄·체감 최대) — tool-call 강한 모델 확인. *없으면 L1–L4가 다 묻힌다.*
2. **NL 로그/이벤트 검색** (S·읽기전용) — 진단/이해 체감 급상승, 안전 0.
3. **능동 내레이션** (S–M·배선) — "먼저 말하는" 비서 느낌. (b)와 짝.
4. **대화 기억 RAG** (M) — L4 심화, "지난번처럼"이 됨.
5. **조건부 자율 트리거** (M–L) — 자율의 다음 계단. 안전 스냅샷 구독.
6. **음성** (L·폴리시).

> 1~3은 "체감이 똑똑해지는" 묶음(저난이도·고체감), 4~5는 "실제로 더 자율"인 묶음.

---

## 6. 정확도 리스크 완화 (SWT 70% 교훈 — 유지)

1. **작고 명확한 도구셋 + 엄격한 JSON 스키마**(function-calling 신뢰도↑).
2. **세계를 바꾸는 행동(goto/expose/run) 전 항상 확인 단계** — 오실행을 사람이 차단.
3. **안전 게이트가 위험 오실행을 물리적으로 차단**(정확도와 독립).
4. 모든 호출을 `ActionLog`에 기록 → 실패 패턴 분석·도구 개선 루프.

## 7. 한계 인지

- (b)빈틈("학습하는 운영 관측소 부재")은 *부재의 증거* — 비공개 시스템 배제 못함. 2026-06 스냅샷.
- **function-calling 신뢰도가 진짜 난관.** 안전은 게이트로 막히지만 "의도대로"는 모델·도구 설계에
  계속 공들일 것. (그래서 우선순위 1번이 모델 점검.)
- MCP 옵션: ASTERION API를 MCP 서버로 노출하면 어떤 LLM 클라이언트든 드라이버가 된다(저결합,
  pixinsight-mcp 인접 선례). 임베디드 챗과 병행 가능.
